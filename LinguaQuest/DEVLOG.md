### What was built

#### Indoor NPC System (Phase 3 partial)
Full indoor NPC interaction from scratch:

**State variables added:**
- `indoorNPCPX / indoorNPCPY` — NPC pixel position (set on `enterHouse`, upper-center of room)
- `indoorWalkFrame` — walk animation frame (0–5), cycles at ~10fps when moving
- `indoorWalkTick` — frame accumulator for walk animation timing

**`drawSpriteFromSheet(ctx, texKey, row, col, x, y, TS)`**
- Reusable helper: draws one frame from a 32×32 spritesheet at the right row/col
- Used by both player and NPC rendering

**`PLAYER_SS_ROW` map** — `{down:0, left:1, up:2, right:3}` — matches actual player_ss spritesheet layout

**`drawIndoorPlayer` — now directional + animated:**
- Reads `indoorLastDir` → picks correct spritesheet row
- Reads `indoorWalkFrame` → picks correct column (0 = idle, 1–5 = walk cycle)
- Uses `drawSpriteFromSheet` with early return; falls back to old colored-rect on failure

**`drawIndoorNPC(ctx, px, py, TS, npc)`**
- Draws NPC sprite (player_ss row 0, col 0 = face-down idle)
- Draws NPC name label above sprite (dark background pill, green text)
- Falls back to colored circle+body if spritesheet unavailable

**Depth sorting in `drawIndoor()`:**
- NPC drawn before player if NPC is higher on screen (lower Y), after if lower — correct overlap

**Walk animation in animation loop:**
- Direction tracked from `vx/vy` each frame → `indoorLastDir` updated
- `indoorWalkTick += dt*60` at 60fps → frame advances every 6 ticks (~10fps)
- Idle resets both `indoorWalkFrame` and `indoorWalkTick` to 0

**NPC proximity hint:**
- In animation loop: computes `Math.hypot(indoorPX-indoorNPCPX, indoorPY-indoorNPCPY)`
- Within 2.5 tiles (120px): shows `[E] Talk` interact prompt above NPC on screen
- Uses `canvas.getBoundingClientRect()` + scale factor to position HTML prompt over canvas NPC
- Hides prompt outside range and on `exitHouse()`

**`INDOOR_KEY_DOWN` — smart E key:**
- If near NPC (`dist < INDOOR_NPC_TALK_DIST = 2.5 * INDOOR_TS`): opens dialogue (or closes if already open)
- If far from NPC: exits house
- No change when dialogue is open and E pressed — falls through to closeDialogue

**`openIndoorNPCDialogue(npc)`**
- Opens the same dialogue box used for outdoor NPCs
- NPC id keyed to `indoorSourceKey` so each house has persistent dialogue history
- Shows NPC portrait (player sprite frame 0), name, personality tag, greeting in target language
- Full `sendDialogue()` / translation reveal / typing indicator all work automatically

### Architecture notes
- Indoor NPC position: `(cols/2, max(2, floor(rows*0.3)))` — upper-center, avoids wall row 0
- NPC data lives in `indoorData.npc` (generated by Claude with name, greeting, ai_context, vocab_focus)
- All indoor dialogue reuses the same `sendDialogue()` + `dialogueHistory` system as outdoor

## Session 4 — 2026-03-13

**Goal:** Architecture & design session — full game redesign before next build phase

### No code written this session. All design.

### What was decided

**Fundamental direction change:** LinguaQuest is no longer a sandbox world-generator. It's a **narrative detective RPG** with AI-driven living NPCs. The world prompt bar is being removed. World generation now happens through a campaign system.

### Default Campaign Designed: "La Lengua del Culpable"
- Player arrives to deliver a letter to Don Esteban Velarde — a merchant who has vanished
- Coastal village of Arenas Blancas; mystery deepens eastward across 3 acts
- Main cast of 6: Petra (innkeeper), Ramos (fisherman), Father Tomás (priest), Mirela (constable), Sonia (niece), Delgado (villain)
- Truth: Don Esteban was smuggled out by Los Silenciosos before he could testify
- Resolution: rescue him from La Hacienda Roja two regions east

### Living NPC System Designed
Every NPC has a **Soul Document** — not a quest queue, but a character: history, fears, desires, contradictions, speech pattern, current preoccupation, secrets, faction ties, emotional state. Goals emerge organically from the soul meeting world circumstances.

Key decisions:
- **Reasoning loop fires once per encounter** (when player talks to NPC, and on chunk entry sweep) — catches up on elapsed in-game time, derives what they've been doing
- **Cascade writes** — when NPC A's loop says they talked to NPC B, NPC B's event log gets written immediately (no extra Claude call). NPCs know things without the player being the messenger.
- **Any NPC talked to gets a full soul document** — generated in background during first conversation. Promoted from ambient to fully simulated.
- **NPCs have lives, not quest queues** — goals are always derived from soul + world state, never pre-authored or auto-generated on quest completion

### World Architecture Designed (4 systems)
1. **Soul Documents** (`lq_npcs.soul`) — fixed-length, updated in place
2. **NPC Event Logs** (`lq_npc_event_logs`) — append-only personal history, read as rolling window
3. **World Event Log** (`lq_world_events`) — objective spatial record of physical events; used by chunk loading to render current reality
4. **Planned Event Queue** (`lq_planned_events`) — decisions already made, waiting to execute

### World Tick Designed
- `setInterval` every 10 real seconds (only while player is active, foreground window)
- Advances `lq_planned_events` — pure state machine, almost never needs Claude
- Template D (Claude) fires only when a resolved event requires a new NPC decision
- **Idle detection:** 2-minute threshold + Page Visibility API → pause overlay → time stops
- **Time only passes while player is actively playing** — no between-session advancement

### 4 Prompt Templates Defined
- **Template A** — Soul Generation (600 tokens max)
- **Template B** — Reasoning Loop (500 tokens max)
- **Template C** — Conversation, streamed (300 tokens max)
- **Template D** — World Tick Resolution (250 tokens max)
